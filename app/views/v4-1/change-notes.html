Changes Notes for v4.1

  • Added new flow for returning the borrower to the enter their token page and to display an error message when the borrower enters either their borrower token or their date of birth incorrectly. This differs from the recommened GDS approach of informing the user when an error is encountered on each page as we thought this method was more secure as it is the main way in to view the deed. We think this might be confusing for the borrowers though so we will test this in the next user research session.
